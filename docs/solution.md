# Example Document

Place your solution documentation in the `./docs` directory. Any media files that you want to include in your documentation, such as images, audio files or recordings, should be placed in the `./docs/media` directory.

It is best to create and maintain Markdown documentation within a Git repository.<br>
See below for some useful references.

* [Use Markdown in Azure DevOps][1]
* [The Markdown Guide][2]
